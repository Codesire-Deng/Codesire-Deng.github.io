1. 自我剖析

1.1 简介

每阵起舞的风都蕴含阳光的注入，每项宏大的工程都起草于稿，每个独特的人都经由小处的雕刻。从自己身上也许读不出惊天泣鬼的史诗，但不必担心这种空荡：收获的季节总会来临。

  • 外语：英语四级；六级战备中

  • Codeforces.com Candidate Master（用户名：cj_dzf）
  • 省级、国家级奥林匹克竞赛参赛次数≥20
  • CR机器人（Android）开发者

1.2 职业性格

MBTI职业性格测试

关键字 I N T P
描述 内向 直觉 思考 知觉
寻迹 外表缄默，倾向思考，优先考虑独立解决方案 擅长体会问题的逻辑走向，能深入浅出地表达思路，对墨守陈规有所保留，爱造新轮子 重视逻辑、公正、公平的价值，习惯客观地分析问题，不注重自己的情感，更喜欢成就而不是被欣赏 顽固地坚持乱桌面，计划经常随时间变化，在新掌握的Github上制作职业生涯规划书

INTP 型的人是解决理性问题者。他们很有才智和条理性，以及创造才华的突出表现,内心专心致志于分析问题。他们苛求精细、惯于怀疑。他们努力寻找和利用原则以理解许多想法。 他们喜欢有条理和有目的的交谈，而且可能会仅仅为了高兴，争论一些无益而琐细的问题。只有有条理的推理才会使他们信服。通常 INTP 型的人是足智多谋、有独立见解的思考者。他们重视才智，对于个人能力有强烈的欲望，有能力也很感兴趣向他人挑战。

可能存在的盲点：由于过于注重逻辑分析，INTP类型的人可能不会考虑别人怎么样。如果某件事不合逻辑，他们很有可能放弃它，就算它很重要。建议找出真正在乎的事将帮助自己更真实地对待自己的情感。 

自我评价

长期接触程序竞赛促成了我的理性思维模式，使我更灵活多变，不畏惧复杂的问题，敢于挑战公式，喜欢学习新知识，面对压力时仍能保持冷静思考。但当面对琐碎的日常工作时我可能缺少耐心，容易发生紊乱。在考虑涉及多人的问题时，我应该谨慎权衡逻辑和他人的感受。

可能适应的行业种类：技能型 研究型 软件工程、网页设计、通信工程、电子工程、化工工程、逻辑型 金融分析等等

1.3 职业兴趣

霍兰德职业兴趣测试

你更倾向于从事实际型、研究型、社会型的职业。你具有较强的动手能力，贴近现实、重视物质，不是理想主义者。喜爱具体明确、需要动手操作的工作环境，对需要技术的活动表现出浓厚的兴趣，较不喜欢需要社交、与人接触的活动，多数情况下必须独立解决工作上的问题。擅长观察、分析、判断和推理，适合需要运用复杂抽象的思考能力的任务。

自我评价

我对感兴趣的事情和任务投入的时间相比其他事务多出许多，比如，我感兴趣的三门专业课都满绩了，却因为时间分配的问题挂了某门数学课，弃修了一门评价特别棒但不感兴趣的通核。而这三门专业课都是指向实际型、研究型工作，如程序员，软件工程师。此外，我对竞赛的延伸工作也充满好感，例如算法工程师，我曾为一道道算法问题经历过无数次的废寝忘食。 上图的推荐职业也许不太适合我，可能是被社会型带偏了

1.4 职业价值观

小结

在我的价值观里，确实更看重工作中的智力发挥空间，渴望工作能带给自己在日常中做小发明创造的机会，而这会会对工作空间的独立自主性提出较大挑战。

2.3 小结

计科/软工专业的就业选择多样，在中低层次，就业门槛平滑，若具有科技行业、互联网行业内中位水平，可得到可观的待遇。多数职位工作强度较高。

在高水平层次，就业门槛突增至博士，对数学水平要求尤其严苛；初次就业时就业者需要用较多时间（以年计）完成项目，有了历史成果之后跳槽现象较为明显。

综上，可得到进修硕博的强烈建议。

3. 职业决策

3.1 目标锁定

目标方案 预留方案 替代方案1 替代方案2
Android应用工程师 AI方向算法工程师 iOS应用工程师 C++Qt方向工程师
熟练设计模式。熟练Spring、Redis、SQL等简单常用框架和工具。学习使用图形引擎。学习网络的基础知识。学习Linux系统高级使用技巧。系统学习Java实现原理。 独立重修线性代数，深入研究统计和概率，全面了解回归。学习现有机器学习模型。展开式理解神经网络，了解NLP等不同域中的AI，熟悉大数据基础知识。掌握优化技术，参加数据科学比赛，阅读大量研究论文，出版论文。调整数学来推广自己的算法。 掌握Xcode的使用，学习Objective-C、Swift的运用。学习iOS运行机理，理解iOS各类API。 了解Windows系统运行机制，掌握Qt各类API的使用。学习网络的基础知识。学习SQL等常用数据库。

3.2 决策分析

SWOT自我分析

S（优势）：具有长期算法竞赛积淀，算法知识宽广而深入，思维灵活；编程语言多样、掌握可靠；有很强的创新能力和竞争意识 W（劣势）：与人交流较少，缺少领导力训练，对简单琐事耐心不足；专业知识缺口仍较大；数学基础没有打好
O（机会）：市场上优质从业者严重供不应求；同水平或高水平竞争对手屈指可数，竞争压力小；有浙江大学镀金敲门砖 T（挑战）：中高层次岗位对专业知识要求极高；需要提高人际交往技能，争取晋升管理类岗位；大学进修机会珍稀，需要在各个不擅长的方面下重功夫

方案敲定过程

考虑自身算法竞赛优势，我首先注意到算法工程师这个业内地位较高、待遇较好、人才供应最紧缺的职业。但观察已就职算法工程师背景发现他们都拥有多年研究数学经验，并持有博士头衔，说明算法工程师和其他计算机类职位存在悬殊的就业门槛差距。如果将算法工程师作为首要目标，我必须走完大赛获奖、读硕攻博的必经路线；如果有一环失败，我可能会陷入普通工程能力不足、短时间内难以就业的窘境。观察在校期经历相仿的学长的选择，发现他们会在学习的过程中兼顾软件工程知识的了解，以保证就业安全。

针对软件工程，我对Android应用的了解较多且有一定的工作经验和成果，对比同龄人而言是一大优势，而且尤其清楚Android应用开发的学习路线，还有在职前辈@葛晓艺老师指导，选择Android方向无疑是最安全稳妥的方案，但代价是待遇上限比不上算法工程师。

最后，由于iOS和C++Qt在学习路线套路上相近于Android，但难度曲线更加简单平滑，我将其摆在替补方案位置。其代价是业内地位和待遇相对较低。预计用到替补方案的概率很小。

4. 计划路径

考虑到不确定性由近及远递增，计划的分辨率将递减。

4.1 近期路径

在这里我列出了能够观测的年内要完成的事。

  • 大一夏学期（含暑假）目标
    • 在7月15之前的个人赛前顺序补充竞赛进度表（见附件）
    • 每天取得充足而科学的休息，(保证11:45pm-7:00am休息)
    • 取得八月区域赛资格
  • 大二目标
    • 随队训练，按顺序完成竞赛进度表
    • 取得至少两场省级以上赛事资格
    • 上学期重修微积分甲1，线性代数；下学期重修微积分甲2。近三年期末考真题达到平均90分。

4.2 中期计划

在这里我列出了本科要掌握的技能。

  • 体验外包
    • 随课程的正常进行，在大三将有能力承接初级的外包工作。
    • 平台：程序员客栈
    • 佣金目标5000元，按市场最低价估算，需要工作十天。
    • 好评率达到80%。
  • 实现《CR视觉机器人》
    • 在《CR机器人》的基础上增加视觉功能，使其识别手机屏幕出现的人物单位。（需要用到TensorFlow入门知识）
  • 随课程补充没涉及到的而在方案中的知识。

4.3 远期计划

  • 考研
    • 若成功，则攻博
  • 按照方案走，初次就业超过一年后应该考虑跳槽
    • （能不能跳槽是检验能力的直观因素）
